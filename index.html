the-smallest-natural-number-which-not-exists-in-list

阅读了一些函数式编程的书籍与面向对象的编程书籍之后，感觉依然只是对函数式有了一个浅显的认识。而买了《函数式算法设计珠玑》已经很久，既然打算写博客，因此打算每一章节写一个自己的认知之类的。（这本书总计30章，彼此之间没有联系，计算一周写一章相关的系列）

问题概述

集合X是一组无序、互异数字组成的列表，设计一个算法以线性时间的结算求解，不在该列表中的最小自然数。

分治算法

分治算法十分常见且有效，他的基本思想是用 minfree xs 和 minfree ys 表示 minfree （xs ++ ys） ，
其中 us\\vs 表示的是从 us 删除 vs的元素。

$$
\begin{aligned}
& (as ++ bs) \setminus\setminus cs = (as \setminus\setminus cs) ++ (bs \setminus\setminus cs) \\
& as \setminus\setminus (bs ++ cs) = (as \setminus\setminus bs) \setminus\setminus cs \\
& (as \setminus\setminus bs) \setminus\setminus cs = (as \setminus\setminus cs) \setminus\setminus bs
\end{aligned}
$$

这里的属性与集合的运算类似（并集、差集的操作），现在假设 as 和 us 是不相交的， bs 和 vs 是不相交的，那么根据以上3条性质推理有：
$$
\begin{aligned}
(as ++ bs) \setminus\setminus (us ++ vs) = (as \setminus\setminus us) ++ (bs \setminus\setminus vs)
\end{aligned}
$$

简单的性质带入即可推导出：

$$
\begin{aligned}
(as ++ bs) \setminus\setminus (us ++ vs) & = (as \setminus\setminus (us ++ vs)) ++ (bs \setminus\setminus (us ++ vs)) \\
& = ((as \setminus\setminus us) \setminus\setminus vs) ++ ((bs \setminus\setminus us) \setminus\setminus vs) \\
& = (as \setminus\setminus vs) ++ ((bs \setminus\setminus vs) \setminus\setminus us) = (as \setminus\setminus vs) ++ (bs \setminus\setminus us)
\end{aligned}
$$

因此，选择任意自然数b， 令 as=[0,.b-1] 以及bs=[b..]，进一步令vs=filter(<b)xs, vs=filter(>=b)xs，所以as 和 vs 不相交， bs和us不相交，所以：

$$
\begin{aligned}
([0…]) \setminus\setminus xs = ([0..b-1] \setminus\setminus vs) ++ ([b..] \setminus\setminus vs)\;where\;(us, vs) = partion\;(<b)\;xs
\end{aligned}
$$

所以可以推导得到：
$$
\begin{aligned}
minfree\;xs\;=\;if\;null([0..b-1]\;\setminus\setminus\;us)\;then\;head([b..] \setminus\setminus vs)\;else\;head([0..] \setminus\setminus us)\;where\;(us, vs) = partion(<b)\;xs
\end{aligned}
$$

下面的问题是如何高效的计算null([0..b-1)\\us，如果直接计算的话，那么消耗的时间将与us的长度有关。因为假设输入的数组内部元素互异，且us中的元素均小于b，所以可以的到

$$
\begin{aligned}
null\; ([0..b-1]\setminus\setminus us) \equiv length\; us == b
\end{aligned}
$$

所以:

$$
\begin{aligned}
minfree\; xs\; = minfree\; 0\; xs
\end{aligned}
$$

$$
\begin{aligned}
minfree\; a\; xs\; & | null\; xs = 0 \\
& | length\; us == b - a = minfree b vs \\
& | otherwise = minfree a us \\
& where\; (us, vs)\; = partion(<b) xs
\end{aligned}
$$

所以现在的问题在于如何确定b的值。b满足要求 b>a ，同时需要us和vs的长度均尽可能的小，所以取b:
$$
\begin{aligned}
b = a + 1 + n\;div\;2
\end{aligned}
$$

最终表达式：

$$
\begin{aligned}
minfree xs = minfree 0 (length xs, xs)
\end{aligned}
$$

$$
\begin{aligned}
minfree a (n, xs) &\;| n == 0 = a \\
&\;| m == b - a = minfree\; b (n-m, vs) \\
&\;| otehrwise = minfree\; a (m, us) \\
&\;where (us, vs) = partion\;(<b)\; xs \\
&\;b = a + 1 + n\;div\;2 \\
&\;m = length\; vs
\end{aligned}
$$

自己的一些认知

函数式算法给我最大的认知是，我们需要去发现问题的基本性质与性质组合后的性质，然后通过对问题的性质描述去解决问题。即我们实际上描述的是：这个问题是什么（what）。而面向对象通常描述的是：这个问题怎么做（how）。第一次在书本上看到是困惑与不解，现在想想深以为然。

install-mathjax-and-set-google-analysis-config

add mathematical equations

原本的hexo并不支持latex形式的数学公式的书写，但是通常学术一些的博客中都会使用到大量的公式。
因此设法可以使用latex形式的数学公式便十分重要了。

安装hexo-renderer-mathjax

npm install hexo-renderer-mathjax –save

由于一些奇怪的原因，这个module无法直接使用，需要做一些修改

./node_modules/hexo-renderer-mathjax/mathjax.html

将script中对应的src修改为：

“https://cdnjs.cloudflare.com/ajax/libs/mathjax/2.7.1/MathJax.js?config=TeX-MML-AM_CHTML"

./nodes_modules/marked/lib/marked.js

1.

before: “escape: /^\(\`*{}[\#$+-.!_>])/,”
after: “escape: /^\(`*[\# +-.!_>])/,”

2.

before: “em: /^\b_((?:__|[\s\S])+?)_\b|^*((?:**|[\s\S])+?)*(?!*)/,”
after: “em: /^*((?:**|[\s\S])+?)*(?!*)/,”

tips

即使安装并修改了上述模块，依然无法在markdown文件中直接使用latex方式书写公式，需要在开始的配置处添加配置项：

mathjax: true

add google analytics

就我使用的hexo版本而言，许多国内的博客教程是错误的。因此略微记录一下。

修改的_config.yaml

添加这个插件确实需要修改配置文件，但是这个配置文件是对应额theme的配置文件，而不是我们一般的博客的配置。

修改_config.yaml方式

这里许多国内的博客介绍的是将google analytics 当做父级，然后添加id， site， enable 等等。
但是似乎并没有生效，去google上检索， 将u-id直接放置于google analytics后面即可。

google_analytics: u-id

monad-related

monad

概念定义

a monad is just a monoid in the category of endofunctors.(单子说白了不过就是自函子范畴上的一个幺半群而已)

前置知识

Binary operation(二元操作)

as its name.

Magma(wiki)

in abstract algebra, a magma(or groupoid: not to be confused with groupoids in category theory) is a basic kind of algebraic structure.
Specifically, a magma consists of a set equipped with a single binary operation.
The binary operation must be closed by definition but no other properties are imposed.
支持二元操作符, 同时该操作满足闭包性质.

Associative(结合律)(wiki)

formally, a binary operation * on a Set S is called associative if it staisfies the associative law:

(x y) z = x (y z) for all x, y, z in S

here, * is used to replace the symbol of the operation, which may be any sumbol, and even the absence of symbol(juxtaposition) as for multiplication.

(xy)z = x(yz) = xyz for all x, y, z in S

the associative law can alse be expressed in functional notation thus: f(f(x, y), z) = f(x, f(y, z))

Semigroup

a semigroup is a set on which the following is defined:

operation: x ◦ y

and on which the following axiom holds:

x ◦ ( y ◦ z ) = ( x ◦ y ) ◦ z associativity

Identity(个人理解)

这个性质指的是,在一集合中,该集合中存在着一单位元素e, 任一元素与该元素执行二元操作得到的结果均为自身.
(关于相等, 二元操作, 顺序就不一一介绍了)

Monoid(wiki)

a monoid is a set on which the following are defined:

operation: x ◦ y
constant: e

and on which the following axioms hold:(associativity, identity)

x ◦ ( y ◦ z ) = ( x ◦ y ) ◦ z
x ◦ e = e ◦ x = x

Inverse

the operation x^(-1) is the inverse operation: applying the operation to an item and its inverse results in the identity element, as the last axiom shows.

Group

a group is a set on which the following are defined:

operations: x ◦ y, x^(-1)
constant: e

and on which the following axioms hold:(associativity, identity, cancellation)

x ◦ ( y ◦ z ) = ( x ◦ y ) ◦ z
x ◦ e = e ◦ x = x
x ◦ x^(−1) = x^(−1) ◦ x = e

Commutative(交换律)(wiki)

the term “commutative” is used in serveral related senses.

  1. a binary operation * on a set S is called commutative if:

x y = y x for all x, y in S

an operation that does not satisfy the above property is call non-commutative

  1. One says that commutes with y under * if:

x y = y x

  1. A binary function: f: A * A -> B is called commutative if:

f(x, y) = f(y, x) for all x, y in A

(这也意味着交换律与闭包无关)

category(范畴)(wiki)

there are many equivalent definitions of a category. One commonly used definition is as follows. a category C consists of:

a class ib(C) of objects
a class hom(C) of morphisms, or arriwsm, or maps, between the objects. each morphism f has a source object a and a target object b where a and b are in ob(C). we write f: a -> b, and we say “f is a morphism from a to b”.

we write hom(a, b)(or hom_c(a, b) when there may be confusion about to which category hom(a, b) refers) to denote the hom-class of all morphisms from a to b.(some authors write Mor(a, b) or simply C(a, b) instead.)
such that the following axioms hold:

(associativity) if f: a -> b, g: b -> c and h: c -> d then h ◦ (g ◦ f) = (h ◦ g) ◦ f, and
(identity) for every object x, there exists a morphism 1_x: x -> x

(some authors write id_x) called the identity morphism for x, such that for every morphism f: a -> x and every morphism g: x -> b, we have 1_x ◦ f = f and g ◦ 1_x = g
from these axioms, one can prove that if there is exactly one identity morphism for every object. some authors use a slight varation of the definition in which each object is identified with the corresponding identity morphism.

morphisam(态射)(wiki)

a category C consists of two classes, one of objects and the other of morphisms.
there are two objects that are associated to every morphism, the source and the target.
for many common categories, objects are sets(often with some additional structure) and morphisms are functions from an object to another object. therefore, the source and the target of a morphism are called domain and codomain respectively.
morphisms are equipped with a partial binary operation, call composition. the composition of two morphisms f and g is defined if and only if the target of f is the source if g, and is denoted g ◦ f(or, sometimes, simly g f).
the source of g ◦ f is the source of f, and the target of g ◦ f is the target of g. the composition statisfies two axioms:
identity: for every object X, there exists a morphism id_x: X -> X called the identity morphism on X, such that for every morphism f: A -> B we have id_B ◦ f = f = f ◦ id_A.
associativity: h ∘ (g ∘ f) = (h ∘ g) ∘ f whenever the operations are defined, that is when the target of f is the source of g, and the target of g is the source of h.

functor(函子)(wiki)

let C and D be categories. a functor F form C to D is a mapping that:

  1. associates to each object X in C an object F(X) in D,
  2. associates to each morphism f: X -> T in C a morphism F(f): F(X) -> F(Y) in D such that the following two conditions hold
  3. F(id_x) = id(F(x)) for every object X in C,
  4. F(g ◦ f) = F(g) ◦ F(f) for all morphisms f: X -> y and g: Y -> Z in C.

That is, functors must preserve identity morphisms and compostition of morphisms.

endofunctor(wiki)

a functor that maps a category to itself.

结语

如果所有的前置知识都可以正确理解的话,那么正如概念描述中所述:a monad is just a monoid in the category of endofunctors.
